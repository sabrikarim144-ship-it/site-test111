Plan d'action Scrum pour tester le site Test Mastery Platform ISTQB

Durée totale

4 sprints

2 semaines chacun

Environnement

UAT

GitHub Pages + Xray

Test fonctionnel
Test visuel
Collaboration équipe
Sprint 0 — Semaine 1-2 Mise en place & prise en main

Objectif du sprint

Toute l'équipe connaît le site, Jira/Xray est opérationnel, les rôles sont définis.

User stories / tâches

Définir les rôles Scrum : Scrum Master, PO, testeurs Équipe
Créer le Test Plan dans Xray et assigner les membres Équipe
Explorer le site ensemble et lister toutes les pages/sections Fonctionnel
Définir les critères visuels attendus (couleurs, mise en page, responsive) Visuel
Sprint 1 — Semaine 3-4 Tests de la navigation & contenu statique

Objectif du sprint

Valider que toutes les pages s'affichent correctement et que la navigation fonctionne.

User stories / tâches

Tester les liens du menu : Accueil, Types de Tests, Tests Statiques, Tests Dynamiques, Quiz Fonctionnel
Vérifier l'affichage des 4 tableaux ISTQB (colonnes, alignement, lisibilité) Visuel
Tester les 5 cartes cliquables "Types de Tests" (Boîte noire, Boîte blanche…) Fonctionnel
Vérifier l'affichage de la pyramide des tests et le texte au clic sur chaque niveau Visuel
Sprint 2 — Semaine 5-6 Tests du Quiz interactif

Objectif du sprint

Couvrir tous les scénarios du quiz : filtres, nombre de questions, résultats.

User stories / tâches

Tester les filtres par thème : Tests Statiques, Dynamiques, Niveaux, Boîtes N/B Fonctionnel
Tester les 4 options de nombre de questions (5, 10, 15, 20) et vérifier le compte Fonctionnel
Vérifier l'affichage du score (0/0, pourcentage) et le bouton Recommencer Visuel
Tester la combinaison filtre + nombre de questions (ex: Boîtes N/B + 10 questions) Fonctionnel
Vérifier le bouton "Lancer le quiz ↗" avec différentes combinaisons Fonctionnel
Sprint 3 — Semaine 7-8 Tests avancés & rapport final

Objectif du sprint

Tests responsive, accessibilité, rapport de bugs et retrospective d'équipe.

User stories / tâches

Tester l'affichage responsive sur mobile, tablette et desktop Visuel
Vérifier la lisibilité (contraste texte/fond, taille des polices) Visuel
Rédiger les rapports de défauts trouvés dans Jira et les lier aux test cases Xray Équipe
Rétrospective : ce qui a bien marché, ce qui peut être amélioré Équipe
Cérémonies Scrum à faire chaque sprint
Sprint Planning Jour 1 — choisir les user stories du sprint et les assigner
Daily Chaque jour — 15 min : qu'est-ce que j'ai fait / je vais faire / blocage ?
Sprint Review Jour 10 — présenter les tests réalisés et les bugs trouvés
Rétrospective Jour 10 — améliorer le processus pour le sprint suivant
